发表于 2019-11-04 | 分类于 linux | 阅读次数:


select被用来高效的监听多个文件描述符，去看它们是否可读，可写，条件触发。
其中readfds,writefds,exceptfds三个文件描述符集合用fd_set来表示。

这是fd_set的定义，显然select可监听的文件描述符有限制，可以看出默认大小为1024。

简单分析，可以看出先是判断超时时间timeval变量，判断它的有效性。可以看到有两个关键函数，core_sys_select和poll_select_finish。
先来看core_sys_select

其中有个数据结构为fd_set_bits。其中包含六个long*变量，三个为用户变量，三个为内核区变量。关键函数为do_select(),在它之前先把数据从fds从用户区复制到内核去，执行完do_select之后，在把结果复制到用户区。下面看看do_select

这个是核心，我们看中间的无限for循环，退出的情况包括timeout,retval,signal_pending,分别是超时，有fd就绪，有其它信号来临。我们从这个循环可以看出，他是轮询判断的，每一轮判断64个fd(64位机器，每字节8位当然有些并不是要监听的fd),注意到vfs_poll()，发现是直接调用file->f_op->poll(file, pt)，而文件描述符对应的驱动调用自己的poll函数时，会调用poll_wait。查看poll_wait函数发现它是调用pt->_qproc(),也就是wait这个变量。所以wait是一个很关键的所在，wait是poll_table类型结构体，wait = &table.pt, 然后我们查看它的定义(下面),发现是非常简单的,文档写的很清楚，帮助实现f_op->poll。下面我们看wait是如何初始化的，注意到它来自于table的pt，我们看看如何初始化table的，发现调用了poll_initwait()函数，里面有一个init_poll_funcptr()函数，追进去查看，很简单的一个绑定，现在关键是pqroc的实现，发现它来自于/_/_pollwait的实现，查看这个函数，发现主要是绑定pollwake唤醒函数和把wait添加到文件描述符的链表的头结点。
