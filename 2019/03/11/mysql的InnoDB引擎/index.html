mysql的InnoDB引擎

发表于 2019-03-11 | 阅读次数:

InnoDB是一个平衡高可用性和高性能的通用存储引擎.mysql创建表时如果不指定引擎,默认使用InnoDB引擎(mysql8.0).InnoDB有四个主要的优点(来自于mysql官方文档):

  • 它的DML操作支持ACID特性,运用事务特性去保护用户的数据.
  • 行级别的锁和oracle风格一致性读写可以增加多用户的并发性能.
  • InnoDB的表会整理硬盘上的数据去优化基于主键的查询,每个表都有一个聚集索引让主键查询更加高效.
  • InnoDB支持外键约束,可以确保数据的完整性.

Innodb一些额外的特点:

  • 当服务器由于断电崩溃时,无论MySQL处于什么状态,当你再次启动时,它都会恢复到最后一次提交事务的状态
  • 它在内存中有一个自己的数据缓冲池,频繁访问的数据会被放到缓冲迟中
  • innodb不允许并发同时读写一个数据表
  • 自适应哈希索引
  • 它所能处理的一个表的容量高达2GB

InnoDB的ACID模型

  • 原子性(atomicity) 事务是原子性的,要么成功要么失败,主要通过commit, rollback操作来体现
  • 一致性(consistency)InnoDB的特性体现在双写缓存和崩溃恢复.
  • 隔离性(isolation)可以通过设置每个事务的隔离级别来体现
  • 持久性(durability)InnoDB在在这方面依赖于具体的硬件配置.事务完成之后,仍然会保存事务的记录到非易失性存储器当中

应用场景
显然由于InnoDB支持事务操作,所以对表的数据一致性和完整性要求比较高的应该使用InnoDB引擎.

聚集索引
mysql的InnoDB引擎支持聚集索引.其实聚集索引和表的主键是一个意思.

  • 当创建基于InnoDB的表时,若定义了主键,那么InnoDB会使用它来作为聚集索引(一般来说都用自增的列来作为主键).如果创建表时没有指定主键,mysql会自动找一个非空并且有唯一值的列作为主键.当然也有可能找不见这样的列,这种情况下mysql会自动生成一个隐藏列
    InnoDB的聚集索引是通过B树来实现的.
