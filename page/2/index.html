cpp复习笔记

发表于 2019-03-29 | 阅读次数:

c++中存储期分为四大类,分别是自动存储期,静态存储期,线程存储期,动态存储期.

  • 自动存储期 对象的存储在外围代码开始时分配,结束时解分配.(除了声明为static, extern, thread_local的所有局部对象拥有存储期)
  • 静态存储期 对象的存储在程序开始时分配,程序结束时解分配.所有声明于命名空间作用域的对象加上static和extern的对象拥有此存储期.
  • 线程存储期 线程里面的分配对象(thread_local)
  • 动态存储期 动态内存分配(new)

static可以被用来声明以下三种形式,通俗点来说就是普通变量,普通函数,成员变量,成员函数

  • 声明有静态存储期和内部链接的命名空间成员(namespace)
  • 定义有静态存储期并且仅仅初始化一次的块作用域变量({}中)
  • 声明不绑定到特定实例的类成员(类的静态变量和静态函数) 静态成员函数不能为virtual,const,volatile

http相关

发表于 2019-03-21 | 阅读次数:

http主要版本分为0.9,1.0,1.1,2.0;
目前广泛使用的是http1.1.这一版本协议默认支持持久连接,对于http1.0来说,通常是在请求的包头添加Connection: Keep-Alive.持久连接主要是使用同一个TCP连接发送和接收多个http请求和应答.默认情况下服务器端会对连接时间有个限制.
http请求的格式: 请求行(方式 url 版本),请求头,空行,其他消息体.
http应答的格式:类似与请求的格式,不过请求行是由版本号,状态码,状态解释组成.
http压缩的通常方式时gzip和deflate,http请求会在请求头通过Accept-Encoding方法通知服务器自己可以接收什么压缩方式,http应答会通过Content-Encoding或者Transfer-Encoding来表明使用的压缩方案.

DNS协议详解

发表于 2019-03-15 | 阅读次数:

DNS协议是一个重要的协议.通常所说的dns查询都是基于udp的.udp报文在网络中的合适最大报文长度为512字节(576-64),576为rfc建议的实际网络中的最小MTU值,64为ip报文最大报头(60+4).当dns数据超过512字节时.如果过服务器端和客户端都支持EDNS协议,则使用它.否则用tcp协议来承载数据.
DNS查询有递归,非递归,迭代三种方式,通常这三种模式是交互使用的.非递归是利用了缓存的原理,先查询dns缓存,直接从缓存返回数据.
递归的话值主机查询默认的DNS服务器,如果服务器上没有对应的数据,那服务器去继续查询直到返回结果,再把结果返会给主机.
迭代的话当默认DNS服务器查询时可以先查询根域名服务器,返回对应的顶级域名服务器地址,默认服务器再从顶级域名服务器查询结果,这种迭代式一级一级域名的查询就是迭代查询.
MTU数据链路层的帧的数据长度最大值为1500,最小应该为576,两个单位都是字节
MSS代表TCP最大报文段长度,通常在tcp三次握手时确定

mysql的InnoDB引擎

发表于 2019-03-11 | 阅读次数:

InnoDB是一个平衡高可用性和高性能的通用存储引擎.mysql创建表时如果不指定引擎,默认使用InnoDB引擎(mysql8.0).InnoDB有四个主要的优点(来自于mysql官方文档):

  • 它的DML操作支持ACID特性,运用事务特性去保护用户的数据.
  • 行级别的锁和oracle风格一致性读写可以增加多用户的并发性能.
  • InnoDB的表会整理硬盘上的数据去优化基于主键的查询,每个表都有一个聚集索引让主键查询更加高效.
  • InnoDB支持外键约束,可以确保数据的完整性.

Innodb一些额外的特点:

  • 当服务器由于断电崩溃时,无论MySQL处于什么状态,当你再次启动时,它都会恢复到最后一次提交事务的状态
  • 它在内存中有一个自己的数据缓冲池,频繁访问的数据会被放到缓冲迟中
  • innodb不允许并发同时读写一个数据表
  • 自适应哈希索引
  • 它所能处理的一个表的容量高达2GB

InnoDB的ACID模型

  • 原子性(atomicity) 事务是原子性的,要么成功要么失败,主要通过commit, rollback操作来体现
  • 一致性(consistency)InnoDB的特性体现在双写缓存和崩溃恢复.
  • 隔离性(isolation)可以通过设置每个事务的隔离级别来体现
  • 持久性(durability)InnoDB在在这方面依赖于具体的硬件配置.事务完成之后,仍然会保存事务的记录到非易失性存储器当中

应用场景
显然由于InnoDB支持事务操作,所以对表的数据一致性和完整性要求比较高的应该使用InnoDB引擎.

聚集索引
mysql的InnoDB引擎支持聚集索引.其实聚集索引和表的主键是一个意思.

  • 当创建基于InnoDB的表时,若定义了主键,那么InnoDB会使用它来作为聚集索引(一般来说都用自增的列来作为主键).如果创建表时没有指定主键,mysql会自动找一个非空并且有唯一值的列作为主键.当然也有可能找不见这样的列,这种情况下mysql会自动生成一个隐藏列
    InnoDB的聚集索引是通过B树来实现的.

python的new和init

发表于 2019-01-07 | 分类于 python ， 特殊方法 | 阅读次数:

new和init一样都是特殊方法，new的调用发生的更早的一点，new方法应该返回对象的实例，之后会调用init方法，此方法不应该有返回值或者返回值为None
如果new方法没有返回实例，init方法不会触发。new方法通常被用在自定义的类中，比如自定义的类继承自不可变对象（int，string， tuple），比如一个自定义的int类型只能为正。还有一方面python官方提到在元编程中很有用，不过目前不会元编程，下次学时的时候再补上吧
