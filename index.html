mysql事务

发表于 2019-11-12 | 阅读次数:

mysql事务，通常定义为一个最小的不可再分的工作单元，比如转账，付款，事务只和DML（数据操作语言有关）
事务包括四大特征，ACID， 原子性，一致性，隔离性，持久性。
mysql事务隔离性包括四种级别：
未提交读(Read Uncommitted)：允许脏读，其他事务只要修改了数据，即使未提交，本事务也能看到修改后的数据值。也就是可能读取到其他会话中未提交事务修改的数据。
提交读(Read Committed)：只能读取到已经提交的数据。Oracle等多数数据库默认都是该级别 (不重复读)。
可重复读(Repeated Read)：可重复读。无论其他事务是否修改并提交了数据，在这个事务中看到的数据值始终不受其他事务影响。
串行读(Serializable)：完全串行化的读，每次读都需要获得表级共享锁，读写相互都会阻塞。
脏读 会读取到别的事务为提交的改变数据
不可重复读 会读取到别的事务已经提交的改变数据
幻读 会读取到别的事务已经提交的新增数据
mysql默认为可重复读，通过一致性读来实现，主要通过mysql多版本技术建立快照来实现。（详细还需要研究）

mysql语句分类

发表于 2019-11-12 | 分类于 mysql | 阅读次数:

SQL语句可以分为五大部分，包括数据定义，数据查询，数据操作，数据控制（权限管理），数据事务五大部分

  • 数据定义
    数据定义就是创建数据库，表，视图，索引。即create database, create table, create view, create index, 同时还包括修改表结构，格式为alter table <表名> add|drop|modify <列名> ,也包括删除语句，drop table, drop view;
  • 数据查询
    数据查询主要包括select语句，其中有常使用的关键字distinct，AVG，GROUP BY， ORDER BY， LIMIT， distinct用来剔除重复数据
  • 数据操作
    数据操作包括插入，删除，修改。即insert into table values|subquery， delete from table where， update table set <column=values> where
  • 数据控制
    grant，revoke
  • 数据事务
    begin，savepoint, commit，rollback
